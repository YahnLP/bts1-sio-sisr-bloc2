## Objectifs Pédagogiques

**À l'issue de cette séance, l'apprenant sera capable de :**

1. **Expliquer** pourquoi un serveur DHCP unique ne peut pas servir plusieurs VLANs sans agent relais
2. **Configurer** `ip helper-address` sur une interface de routeur pour relayer les requêtes DHCP
3. **Déployer** une infrastructure réseau intégrant VLANs, routage inter-VLAN, DHCP Linux, DNS Bind9
4. **Mettre en relation** tous les services configurés (switch → routeur → serveur Linux)
5. **Diagnostiquer** méthodiquement une panne dans une infrastructure multi-services
6. **Démontrer** sa maîtrise de la Phase 2 lors du TP noté évaluatif
